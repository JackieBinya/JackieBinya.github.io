GraphQL vs REST

Historically REST is the accepted standard for designing web APIs. The main advantages of using REST were: REST is stateless meaning every server call is independent of any other network calls made to the server. The server will not persist any state about the queries it receives. It provides a structured way to query resources. The problem REST was intended to be a strict specification used to design APIs but in reality, it was wildly interpreted: most APIs described as being Restful aren’t really Restful, they don’t adhere to the strict specification REST APIs have shown to be too inflexible to keep up with the rapidly changing requirements on the client apps that access them.
